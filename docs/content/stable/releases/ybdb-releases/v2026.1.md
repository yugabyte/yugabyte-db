What follows are the release notes for all releases in the **YugabyteDB** v2026.1 series. Content will be added as new notable features and changes are available in the patch releases of the YugabyteDB v2026.1 series.

For an RSS feed of all release series, point your feed reader to the [RSS feed for releases](../index.xml).

| <div style="width:150px">Feature</div> | Description |
| :-------- | :---------- |
| {{<tags/feature/tp idea="2652">}} [YSQL Distributed Tracing](/stable/launch-and-manage/monitor-and-alert/ysql-distributed-tracing/) | Export YSQL query lifecycle spans (parse, plan, execute, commit, and RPC calls to tablet servers) as OpenTelemetry traces using W3C `traceparent` context. View waterfall traces in backends such as Jaeger, Grafana Tempo, or Honeycomb. <hr style="margin: 0.5em 0;">**To use**: Disabled by default. Add `otel_collector_traces_endpoint` to [allowed_preview_flags_csv](/stable/reference/configuration/yb-tserver/#allowed-preview-flags-csv), then set `otel_collector_traces_endpoint` to your OTLP/HTTP collector URL (for example, `http://<host>:4318/v1/traces`) on all YB-TServers (restart required). Enable tracing per query with a `traceparent` SQL comment or the `yb_dist_tracecontext` configuration parameter. See [YSQL Distributed Tracing](/stable/launch-and-manage/monitor-and-alert/ysql-distributed-tracing/). |
| {{<tags/feature/ea idea="2470">}} [CDC before-image](/stable/reference/configuration/yb-tserver/#cdc-enable-intra-transactional-before-image) support for multi-operation transactions.| Added support for FULL replica identity mode when a row is modified multiple times within a single transaction. <hr style="margin: 0.5em 0;"> **To use**: Disabled by default. Set [cdc_enable_intra_transactional_before_image](/stable/reference/configuration/yb-tserver/#cdc-enable-intra-transactional-before-image) to `true` on all YB-TServers (restart required). |
| {{<tags/feature/ga idea="946">}}[OIDC-based authentication in YCQL](/stable/secure/authentication/oidc-authentication-ycql/) | Support for OIDC token-based authentication for YCQL databases. This allows YCQL database users to sign in to universes using their JSON Web Token (JWT) as their password. <hr style="margin: 0.5em 0;">**To use**: Disabled by default. Set `ycql_use_jwt_auth=true` and configure `ycql_jwt_conf` on YB-TServer (with YCQL auth enabled). Create YCQL roles and use the JWT as the password. See [OIDC authentication in YCQL](/stable/secure/authentication/oidc-authentication-ycql/). |
| {{<tags/feature/ga idea="990">}}[Instant database cloning](/stable/manage/backup-restore/instant-db-cloning/)| Clones are point-in-time copies of your database that are created in seconds regardless of database size. There is no upfront copy cost since they share underlying storage with the source at creation and diverge only as new writes arrive. You can clone from the current state or from any point within the history retention window. Useful for instant data recovery, pre-change safety nets, and isolated development or test environments. <hr style="margin: 0.5em 0;">**To use**: Enabled by default. See [Clone a database](https://docs.yugabyte.com/stable/manage/backup-restore/instant-db-cloning/#clone-databases). |
| {{<tags/feature/ga idea="2158">}}[YSQL Table Inheritance](/stable/explore/ysql-language-features/advanced-features/inheritance/)| YugabyteDB now supports the PostgreSQL `INHERITS` clause in `CREATE TABLE`, allowing child tables to inherit columns and CHECK constraints from a parent table. See [PostgreSQL table inheritance](https://www.postgresql.org/docs/current/tutorial-inheritance.html). <hr style="margin: 0.5em 0;">**To use**: Enabled by default. See [Example](/stable/explore/ysql-language-features/advanced-features/inheritance/#lexample).|
| {{<tags/feature/ga idea="2196">}}[Connection Manager enhancement](/stable/additional-features/connection-manager-ysql/ycm-setup/#configure) | Reduces latency using an authentication passthrough mechanism when acquiring new logical connections.<hr style="margin: 0.5em 0;">**To use**: Enabled by default. Contact {{% support-general %}} about whether setting `ysql_conn_mgr_use_auth_backend` flag is recommended for your workload. |
| {{<tags/feature/ga idea="2472">}}[Connection Manager prepared statement deallocation]. Added support for DEALLOCATE and CLOSE of prepared statements in Connection Manager, preventing errors when a statement is closed and re-prepared. <hr style="margin: 0.5em 0;">**To use**: Enabled by default. |
| {{<tags/feature/ga idea="1841">}}[Immediate Table Reflection in Publication](/stable/additional-features/change-data-capture/using-logical-replication/advanced-topic/#implicit-publication) Newly added tables now instantly reflect in the publication, eliminating common user errors and enhancing data accuracy. <hr style="margin: 0.5em 0;">**To use**: Enabled by default. See [Implicit publication](/stable/additional-features/change-data-capture/using-logical-replication/advanced-topic/#implicit-publication)|
|{{<tags/feature/ga idea="1107">}}[Query Plan Management (QPM)](/stable/launch-and-manage/monitor-and-alert/query-tuning/query-plan-manage/) | Introduced QPM to ensure stable and adaptable query performance by preventing plan regression and automatically detecting optimal new plans.<hr style="margin: 0.5em 0;">**To use**: Enabled by default. Set the `yb_pg_stat_plans_track` configuration parameter to `top` or `all` (default). |
|{{<tags/feature/ga idea="2481">}}DDL completeness. Added support for previously unsupported PostgreSQL DDL constructs, reducing migration friction from PostgreSQL. New support includes: [ALTER ROUTINE](/stable/api/ysql/the-sql-language/statements/ddl_alter_routine/), [ALTER AGGREGATE](/stable/api/ysql/the-sql-language/statements/ddl_alter_aggregate/), and the [XML data type](/stable/explore/ysql-language-features/xml-ysql/). <hr style="margin: 0.5em 0;">**To use**: Enabled by default. |
